# Contributing

VISITOR-OS is currently in early architecture and foundation phases.

## Principles

- Keep the project simple.
- Prefer maintainable code over clever code.
- Do not add business-specific logic to the core engine.
- Do not add dependencies without a clear need.
- Document important decisions in `docs/DECISIONS.md`.

## Before Adding Features

- Confirm the feature belongs in the current roadmap phase.
- Keep modules isolated.
- Update documentation when architecture or behavior changes.
- Never commit secrets, API keys, passwords, or private client data.

## Commit Style

Use clear, descriptive commit messages.

Example:

`Initialize VISITOR-OS repository structure`
