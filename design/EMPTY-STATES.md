# Empty States

Status: UX/UI specification, no implementation.

## 1. Empty State Purpose

Empty states should help the user understand:

- why there is no data;
- what happens next;
- what action can create value.

They should not be generic decoration.

## 2. Dashboard Empty State

When no activity exists:

- confirm setup status;
- show widget installation/configuration checklist;
- explain that conversations and prospects will appear after visitor activity.

Primary action:

- Open configuration.

Secondary action:

- View installation guide later.

## 3. Conversations Empty State

Message:

- No conversations yet.
- The widget may not be installed or traffic has not started.

Actions:

- Check widget configuration.
- View site setup.

## 4. Prospects Empty State

Message:

- No prospects yet.
- Prospects are created when visitors show intent or provide contact details.

Actions:

- Review conversations.
- Improve quick replies/FAQ.

## 5. Follow-ups Empty State

Message:

- No follow-ups due.

Action:

- View all prospects.

Tone should feel reassuring, not empty or broken.

## 6. Search No Results

Show:

- searched term;
- active filters;
- clear filters action;
- suggestions for broader search.

## 7. Exports Empty State

Message:

- No exports created yet.

Action:

- Create export.

## 8. Configuration Empty State

For optional modules:

- explain what the module does;
- show whether it is enabled;
- provide setup action only if module is available.

## 9. Empty State Rules

- One short title.
- One useful sentence.
- One primary action.
- Optional secondary link.
- Avoid fake data.
- Avoid large decorative illustrations in operational screens.
