# Icons

Status: UX/UI specification, no implementation.

## 1. Icon Library

Recommended library:

- Lucide icons.

Reasons:

- coherent style;
- broad coverage;
- lightweight;
- familiar SaaS patterns;
- available in frontend ecosystems.

## 2. Icon Style

- outline icons;
- consistent stroke width;
- 16 px for compact controls;
- 20 px for standard buttons;
- 24 px for empty states or section headers when useful.

## 3. Icon Usage

Use icons for:

- navigation;
- search;
- notifications;
- settings;
- filters;
- export;
- edit;
- delete/archive;
- status hints;
- communication channels;
- follow-up/reminder.

Do not use icons as decoration only.

## 4. Accessibility

- Icon-only buttons require accessible labels.
- Tooltips for unfamiliar icons.
- Do not rely on icon alone for critical status.
- Pair semantic icons with text in dense business contexts.

## 5. Recommended Icon Map

| Concept | Icon style |
|---|---|
| Dashboard | layout/dashboard |
| Conversations | message circle |
| Prospects | users/user round |
| Follow-ups | calendar/check |
| Analytics | chart |
| Exports | download/file down |
| Configuration | sliders/settings |
| Notifications | bell |
| Search | search |
| Help | circle help |
| Hot prospect | flame or trending up with label |
| Warning | triangle alert |
| Success | check circle |
| Error | circle x |
| Archive | archive |
| Privacy | shield |

## 6. Anti-Patterns

- Mixing icon libraries.
- Using filled and outline icons inconsistently.
- Using abstract icons without labels.
- Oversized icons in operational screens.
