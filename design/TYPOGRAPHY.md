# Typography

Status: UX/UI specification, no implementation.

## 1. Font Choice

Recommended font family:

- Inter or a similar modern sans-serif for admin;
- system font fallback for performance;
- widget should use a safe stack and avoid loading heavy font files unless justified.

## 2. Typography Principles

- Clear hierarchy.
- No decorative typography.
- Compact but readable.
- Labels should be precise.
- Numbers should be easy to scan.

## 3. Admin Scale

Suggested scale:

- Page title: 24-28 px, semibold.
- Section title: 18-20 px, semibold.
- Card title: 14-16 px, medium/semibold.
- Body: 14-16 px, regular.
- Table body: 13-14 px, regular.
- Labels: 12-13 px, medium.
- Helper text: 12-13 px, regular.
- KPI value: 28-36 px, semibold.

Do not scale font size with viewport width.

## 4. Widget Scale

Suggested scale:

- Header: 15-16 px, semibold.
- Message text: 14-15 px.
- Quick replies: 13-14 px.
- Input: 14-16 px.
- Helper/privacy text: 12-13 px.

## 5. Weights

Use:

- regular for body;
- medium for labels and table headers;
- semibold for headings and important values.

Avoid heavy bold everywhere.

## 6. Line Height

Guidance:

- body: 1.4 to 1.6;
- compact table rows: 1.3 to 1.4;
- headings: 1.2 to 1.3.

## 7. Text Rules

- Use sentence case for labels where possible.
- Avoid jargon.
- Keep button labels short and action-oriented.
- Do not use placeholder text as documentation.
- Error text must explain recovery.
