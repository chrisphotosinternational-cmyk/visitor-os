# VISITOR-OS Design Principles

Status: UX/UI specification, no implementation.

## 1. Product Personality

VISITOR-OS must feel:

- simple;
- fast;
- professional;
- premium;
- calm;
- precise;
- trustworthy;
- usable without training.

The interface should never feel like a technical control panel. It should feel like a focused commercial cockpit for small businesses.

## 2. Absolute UX Rules

- Important information must be reachable in 3 clicks or fewer.
- Every screen must answer one primary question immediately.
- Every screen must expose one clear primary action.
- Reading should be mostly vertical.
- Avoid unnecessary modals.
- Always preserve context: site, prospect, conversation, date, status.
- Never make the user guess whether an action succeeded.
- Never hide urgent items behind filters.
- Never use technical vocabulary when plain business language is possible.
- Never overload the first screen with configuration.
- Prefer progressive disclosure to dense advanced panels.

## 3. Screen Priority Question

Each screen starts from one question:

| Screen | Primary question |
|---|---|
| Dashboard | What needs my attention now? |
| Conversations | What are visitors asking and which exchanges matter? |
| CRM | Which prospects should I follow up first? |
| Prospect detail | What happened, what do we know, and what is the next action? |
| Settings | What can I safely configure without code? |
| Widget preview | What will visitors experience? |
| Exports | What data do I need and can I export it safely? |

## 4. Information Hierarchy

1. Urgent action.
2. Commercial opportunity.
3. Context.
4. Details.
5. Secondary actions.
6. Advanced metadata.

## 5. Interaction Principles

- Primary actions are always visible above the fold.
- Destructive actions are visually separated and confirmed.
- Inline editing is preferred for small changes.
- Long configuration forms are grouped into short sections.
- Lists should support search, filter, and sort before bulk actions.
- Empty states must teach the next useful action.
- Loading states must preserve layout and reduce anxiety.

## 6. Premium Simplicity

Premium does not mean decorative. It means:

- clear spacing;
- restrained color;
- consistent typography;
- fast interactions;
- quiet surfaces;
- strong alignment;
- precise labels;
- no visual clutter.
