# Core Values

Les valeurs de VISITOR-OS guident les decisions produit, techniques et commerciales.

## 1. Simplicite

La simplicite est une exigence.

Une fonctionnalite doit etre facile a comprendre, utiliser, tester, maintenir et expliquer.

## 2. Configuration Avant Developpement

Le produit doit permettre d'adapter les comportements par configuration avant d'ecrire du code specifique.

La logique metier ne doit pas etre dispersee dans le code.

## 3. IA Sous Controle

L'IA assiste.

Elle ne remplace pas les regles, la connaissance, la validation et la responsabilite produit.

VISITOR-OS doit toujours savoir fonctionner avec un mock ou un fallback.

## 4. Indépendance Fournisseur

Aucun fournisseur ne doit devenir indispensable.

Les providers IA, email, stockage, recherche ou vectorisation doivent rester interchangeables.

## 5. Donnees Client

Les donnees appartiennent toujours au client.

VISITOR-OS doit les proteger, les isoler, les exporter et les supprimer proprement.

## 6. Documentation Obligatoire

Tout module important doit etre documente.

La documentation fait partie du produit, pas de l'administration.

## 7. Tests Obligatoires

Une capacite critique sans test est incomplete.

Les tests protegent la maintenabilite du projet sur plusieurs annees.

## 8. Une Release = Une Capacite

Chaque release doit avoir un objectif unique.

Cette discipline evite les releases confuses et les regressions invisibles.

## 9. Modularite

Les modules doivent etre compréhensibles, interchangeables et desactivables.

Le coeur produit doit rester stable.

## 10. Robustesse Discrete

VISITOR-OS ne doit pas chercher a impressionner par sa complexite.

Il doit inspirer confiance par sa fiabilite, sa clarte et sa coherence.
