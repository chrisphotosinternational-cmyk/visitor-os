# Product Strategy

VISITOR-OS doit evoluer comme une plateforme produit, pas comme une accumulation de fonctionnalites.

## Positionnement

VISITOR-OS est positionne entre :

- un widget de conversation ;
- un mini CRM ;
- un moteur de connaissance ;
- un moteur d'intelligence client ;
- une plateforme SaaS modulaire.

Le produit doit rester plus simple qu'un CRM enterprise et plus intelligent qu'un simple chatbot.

## Strategie Produit

La strategie produit repose sur cinq piliers :

1. conversation ;
2. connaissance ;
3. qualification ;
4. action ;
5. mesure.

Chaque nouvelle capacite doit renforcer au moins un de ces piliers.

## Coeur Produit

Le coeur produit contient :

- widget visiteur ;
- conversations ;
- prospects ;
- configuration metier ;
- Decision Engine ;
- KMS ;
- CRM minimal ;
- notifications essentielles ;
- analytics utiles ;
- RBAC ;
- multi-tenant.

## Modules Optionnels

Les modules optionnels peuvent inclure :

- reservations ;
- paiements ;
- WhatsApp ;
- campagnes email ;
- connecteurs externes ;
- workflows avances ;
- marketplace ;
- API publique avancee.

Un module optionnel ne doit pas alourdir le coeur produit.

## Regle De Decision

Une fonctionnalite entre dans le coeur produit uniquement si :

- elle sert la mission principale ;
- elle est utile a la majorite des organisations ;
- elle peut etre maintenue par une petite equipe ;
- elle respecte l'isolation multi-tenant ;
- elle peut etre testee clairement ;
- elle ne cree pas de verrou fournisseur.

Sinon, elle reste un module.

## Strategie IA

L'IA doit etre sous controle.

VISITOR-OS doit :

- privilegier la connaissance documentee ;
- expliquer les sources ;
- accepter plusieurs providers ;
- fonctionner sans provider IA actif ;
- eviter de coder la logique metier dans les prompts ;
- conserver une strategie de fallback.

## Strategie Marche

Les premiers segments naturels sont :

- hotels independants ;
- studios photo ;
- commerces locaux ;
- agences ;
- independants premium ;
- petites equipes commerciales ;
- sites vitrines avec demandes entrantes.

VISITOR-OS doit d'abord devenir excellent pour les petites organisations avant de viser les grands comptes.
