# Follow-ups

Follow-ups keep CRM work actionable without adding a full task manager.

## Fields

- prospect;
- optional conversation;
- due date;
- reason;
- status;
- author;
- completed date.

## Status

- `pending`
- `completed`
- `cancelled`

## Views

The backend supports:

- today;
- overdue;
- pending;
- completed.

## Rules

Follow-ups are organization-scoped and protected by CRM permissions.
