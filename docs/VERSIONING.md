# Chatbot Studio Versioning

Each Studio supports controlled publication.

Version states:

- draft;
- preproduction;
- published;
- rolled_back.

## Draft

Draft content can be generated by the wizard, document import or manual Knowledge Engine editing.

## Published

Publishing records a version and makes draft knowledge active.

The widget does not need a new script after publication. It reads the active knowledge on each backend response.

## Rollback

Rollback marks a previous version as published and updates the Studio publication pointer.

Sprint 19 stores the version history and publication pointer. Future releases can expand rollback to restore full snapshots of all knowledge, flow and personality records.
