# Long Term Roadmap

Cette roadmap donne les grandes directions de VISITOR-OS jusqu'a 2030.

Elle n'est pas une liste de promesses commerciales.

Elle sert a proteger la coherence du produit.

## RC2 - Document Intelligence

Objectif :

- importer des fichiers reels ;
- extraire le texte ;
- chunker ;
- indexer ;
- rechercher ;
- preparer le RAG sans l'activer.

Statut attendu :

```text
KMS documentaire exploitable.
```

## RC3 - Retrieval & Vector Evaluation

Objectif :

- evaluer pgvector ;
- comparer Qdrant ;
- definir le modele de chunks ;
- preparer embeddings ;
- mesurer couts et performances ;
- garder le Decision Engine decouple.

Statut attendu :

```text
Decision technique claire pour le RAG.
```

## BETA - Produit Utilisable Par Premiers Clients

Objectif :

- stabiliser widget ;
- stabiliser admin ;
- fiabiliser auth/RBAC ;
- ameliorer KMS ;
- finaliser onboarding ;
- documenter installation ;
- definir premiers cas clients.

Statut attendu :

```text
Produit utilisable hors environnement de developpement.
```

## v1.0 - Plateforme SaaS Stable

Objectif :

- deploiement cloud fiable ;
- sauvegardes ;
- monitoring ;
- configuration multi-tenant ;
- CRM essentiel ;
- KMS stable ;
- notifications essentielles ;
- analytics utiles ;
- documentation complete ;
- politique de release.

Statut attendu :

```text
VISITOR-OS peut etre vendu et exploite en production pour de petites organisations.
```

## v2.0 - Ecosysteme Modulaire

Objectif :

- API publique ;
- SDK ;
- marketplace de modules ;
- connecteurs documentaires ;
- connecteurs metiers ;
- modules optionnels ;
- deploiement dedie ;
- preparation on-premise.

Statut attendu :

```text
VISITOR-OS devient une plateforme extensible.
```

## Objectifs A 3 Ans

Architecture :

- coeur stable ;
- modules interchangeables ;
- contracts publics ;
- observabilite fiable ;
- migration documentee.

Communaute :

- documentation publique ;
- exemples de modules ;
- guides developpeurs ;
- templates d'integration.

Marketplace :

- catalogue de modules ;
- installation controlee ;
- permissions explicites ;
- versioning modules.

API Et SDK :

- API publique documentee ;
- SDK JavaScript/TypeScript ;
- webhooks fiables ;
- exemples complets.

Deploiement :

- cloud managé ;
- deploiement dedie ;
- option on-premise evaluee ;
- sauvegardes et restauration documentees.

## Ce Qui Doit Attendre

Ne pas lancer avant validation marche :

- marketplace complete ;
- workflows complexes ;
- mobile natif ;
- SSO enterprise ;
- BI avancee ;
- self-hosted AI infrastructure ;
- connecteurs trop nombreux.
